Write HTML in
pure Python.

Dominate is a Python library for creating and manipulating HTML documents using an elegant DOM API. No template languages. No string formatting. Just Python.

✓ 100% test coverage ✓ Python 3.10+ zero runtime deps LGPL-3.0
$ pip install dominate

Installation

Install dominate from PyPI using pip. No system-level dependencies required.

pip install dominate

Or install from source:

git clone https://github.com/Knio/dominate.git
cd dominate
pip install -e .

Dominate has zero runtime dependencies — it uses only the Python standard library.

Quick Start

A complete HTML document in under 15 lines of Python.

Python
import dominate
from dominate.tags import *

doc = dominate.document(title='Dominate your HTML')

with doc.head:
    link(rel='stylesheet', href='style.css')
    script(type='text/javascript', src='script.js')

with doc:
    with div(id='header').add(ol()):
        for i in ['home', 'about', 'contact']:
            li(a(i.title(), href='/%s.html' % i))
    with div():
        attr(cls='body')
        p('Lorem ipsum..')

print(doc)
HTML Output
<!DOCTYPE html>
<html>
  <head>
    <title>Dominate your HTML</title>
    <link href="style.css"
          rel="stylesheet">
    <script src="script.js"
            type="text/javascript"></script>
  </head>
  <body>
    <div id="header">
      <ol>
        <li><a href="/home.html">Home</a></li>
        <li><a href="/about.html">About</a></li>
        <li><a href="/contact.html">Contact</a></li>
      </ol>
    </div>
    <div class="body">
      <p>Lorem ipsum..</p>
    </div>
  </body>
</html>

Hello, World!

Every HTML element is a Python class. Pass children and text as constructor arguments.

Dominate exposes a class for each HTML element. The constructor accepts child elements, text, or keyword attributes. Nodes return their HTML from __str__, __unicode__, and render().

Python
print(html(body(h1('Hello, World!'))))
Output
<html>
  <body>
    <h1>Hello, World!</h1>
  </body>
</html>

Attributes

Pass keyword arguments to set HTML attributes. Reserved Python keywords have aliases.

Keyword aliases

The attributes class and for conflict with Python reserved keywords. Use any of these aliases:

Python
test = label(cls='classname anothername', fr='someinput')
print(test)
Output
<label class="classname anothername"
       for="someinput"></label>

data-* attributes

Use data_* prefix for custom HTML5 data attributes. Underscores are converted to dashes automatically.

Python
test = div(data_employee='101011')
print(test)
Output
<div data-employee="101011"></div>

Dictionary interface

Attributes can be set after construction using a dictionary-like interface:

Python
header = div()
header['id'] = 'header'
print(header)
Output
<div id="header"></div>

Complex Structures

Use += and .add() to build trees. The .add() method returns children for further chaining.

Python — using +=
lst = ul()
for item in range(4):
    lst += li('Item #', item)
print(lst)
Output
<ul>
  <li>Item #0</li>
  <li>Item #1</li>
  <li>Item #2</li>
  <li>Item #3</li>
</ul>

Iterables

Dominate accepts any Python iterable — including generators — as children:

print(ul(li(a(name, href=link), __pretty=False) for name, link in menu_items))

Tuple unpacking with .add()

When multiple children are passed to .add(), they are returned as a tuple — useful for clean assignment:

Python
_html = html()
_head, _body = _html.add(head(title('My Page')), body())
names = ['header', 'content', 'footer']
header, content, footer = _body.add(
    [div(id=name) for name in names]
)
print(_html)
Output
<html>
  <head><title>My Page</title></head>
  <body>
    <div id="header"></div>
    <div id="content"></div>
    <div id="footer"></div>
  </body>
</html>

Rendering

Control indentation, whitespace, and XHTML output via render() arguments.

By default, render() produces human-readable output with one element per line and two-space indentation. Options propagate to all descendant nodes.

ParameterTypeDefaultDescription
pretty bool True Enable human-readable indented output
indent str ' ' Indentation string (two spaces by default)
xhtml bool False Render self-closing tags as XHTML (<br />)
__pretty bool True Per-element pretty flag set at construction time
Compact output
a = div(span('Hello'))
print(a.render(pretty=False))
Output
<div><span>Hello</span></div>
XHTML mode
d = div()
with d:
    hr(); p("Test"); br()
print(d.render(xhtml=True))
Output
<div>
  <hr />
  <p>Test</p><br />
</div>

Context Managers

Use Python's with statement to build nested trees idiomatically.

Any node created inside a with block that hasn't been explicitly added elsewhere is automatically appended to the context node when the block closes.

Python
h = ul()
with h:
    li('One')
    li('Two')
    li('Three')
print(h)
Output
<ul>
  <li>One</li>
  <li>Two</li>
  <li>Three</li>
</ul>

Nested context managers

Python
h = html()
with h.add(body()).add(div(id='content')):
    h1('Hello World!')
    p('Lorem ipsum ...')
    with table().add(tbody()):
        row = tr()
        row += td('One')
        row.add(td('Two'))
        with row:
            td('Three')
print(h)
Output (abbreviated)
<html><body>
  <div id="content">
    <h1>Hello World!</h1>
    <p>Lorem ipsum ...</p>
    <table><tbody><tr>
      <td>One</td>
      <td>Two</td>
      <td>Three</td>
    </tr></tbody></table>
  </div>
</body></html>

attr() and text() in context

Use attr() to set attributes on the current context node, and dominate.util.text() to add mixed text content:

Python
from dominate.util import text
para = p(__pretty=False)
with para:
    text('Have a look at our ')
    a('other products', href='/products')
print(para)
Output
<p>Have a look at our
  <a href="/products">other products</a>
</p>

Decorators

Use tag classes or instances as function decorators to build reusable widgets.

Instead of the boilerplate pattern of with tag() as t: ...; return t, decorate your function with the tag directly:

Using a class as decorator
@div
def greeting(name):
    p('Hello %s' % name)

print(greeting('Bob'))
Output
<div>
  <p>Hello Bob</p>
</div>

Instance as decorator

Use a tag instance as the decorator to pre-populate the root node with attributes or children. Each call returns a copy.

Using an instance as decorator
@div(h2('Welcome'), cls='greeting')
def greeting(name):
    p('Hello %s' % name)

print(greeting('Bob'))
Output
<div class="greeting">
  <h2>Welcome</h2>
  <p>Hello Bob</p>
</div>

Creating Documents

The document class manages the full HTML boilerplate for you.

A new document() automatically creates the <html>, <head>, and <body> structure:

Python
d = document()
d += h1('Hello, World!')
d += p('This is a paragraph.')
print(d)
Output
<!DOCTYPE html>
<html>
  <head><title>Dominate</title></head>
  <body>
    <h1>Hello, World!</h1>
    <p>This is a paragraph.</p>
  </body>
</html>
ParameterTypeDefaultDescription
titlestr'Dominate'Sets the <title> element in <head>
doctypestr'<!DOCTYPE html>'Doctype declaration prepended to output
requestobjectNoneOptional request object for context
Tip: Nodes added directly to a document (via += or with doc:) are appended to the <body>. Access doc.head, doc.body, and doc.title as properties.

Embedding HTML

Use raw() to inject pre-formed HTML strings without escaping.

By default, strings passed to dominate are HTML-escaped. Use dominate.util.raw to bypass escaping for trusted HTML strings:

from dominate.util import raw

td(raw('<a href="example.html">Example</a>'))
Warning: Only use raw() with trusted input. Passing untrusted user content will expose you to XSS vulnerabilities.

SVG Support

The dominate.svg module mirrors dominate.tags for SVG elements.

SVG elements automatically convert underscores to dashes in attribute names:

Python
from dominate.svg import *
print(circle(stroke_width=5))
Output
<circle stroke-width="5"></circle>

API — dom_tag

Base class for all HTML and SVG elements. All tag classes inherit from dom_tag.

Method / AttributeSignatureReturnsDescription
__init__ (*args, **kwargs) dom_tag Children passed as positional args; attributes as keyword args
render (pretty=True, indent=' ', xhtml=False) str Render the node and all descendants to an HTML string
add (*args) dom_tag | tuple Add children. Returns single child or tuple when multiple passed
__iadd__ (other) dom_tag Append a child via += operator
__getitem__ (key: str) str Get attribute value by name
__setitem__ (key: str, value: str) None Set attribute by name
__enter__ / __exit__ () dom_tag Context manager support for with statement
__str__ () str Equivalent to render() with defaults

API — document

Subclass of html that manages a complete HTML document structure.

MemberTypeDescription
document(title, doctype, request)constructorCreate document. Keyword args set title, doctype declaration, and optional request.
.headhead (dom_tag)The <head> element — use as context manager to add stylesheets, scripts, meta tags
.bodybody (dom_tag)The <body> element — nodes added to the document directly are appended here
.titlestrGet or set the document title string

API — dominate.util

Utility classes for text nodes, raw HTML, and context attribute setting.

NameSignatureDescription
text (content: str, escape: bool = True) A plain text node. Use inside with blocks for mixed text+tag content. Set escape=False for pre-escaped content.
raw (content: str) Inject a raw, unescaped HTML string. Equivalent to text(content, escape=False). Use only with trusted input.
attr (**kwargs) Set attributes on the currently active context node (the innermost with block target).

API — tags & svg

Every standard HTML5 element and SVG element is available as a Python class.

Import all HTML tags with from dominate.tags import * or import individually. All tags inherit from dom_tag and accept the same constructor signature.

Selected HTML tags

dominate.tagsdiv, span, p, a, ul, li, ol, table, tr, td, th, form, input_, button, h1…h6, img, script, link, meta, head, body, html, br, hr, pre, code, em, strong, label, select, option, textarea …All standard HTML5 elements. Note: input is input_ to avoid Python keyword conflict.
dominate.svgsvg, circle, rect, path, line, polyline, polygon, ellipse, g, text, tspan, defs, use, symbol …Attribute underscores auto-converted to dashes (e.g. stroke_width → stroke-width)
